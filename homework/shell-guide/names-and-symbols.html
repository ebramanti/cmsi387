HOME

HOME is the variable notation for the home directory of the user.

LANG

LANG is a variable that determines the locale category for native language, which can be used to determine the language to use for error messages, instructions, etc.

PATH

PATHis an environment variable specifying a set of directories where executable programs are located.

SHELL

mkdir makes a directory (or subfolder) within the current directory.

Example: 

Edwards-MacBook-Air:cmsi387 Edward$ mkdir assignments
Edwards-MacBook-Air:cmsi387 Edward$ ls
README.md assignments homework

The example shows the current directory is cmsi387. When mkdir assignments is inputted into the command line, it creates a subfolder in the current directory (NOTE: mkdir does not output confirmation of directory creation). Therefore, using the command ls that was explained above, we discover that indeed mkdir created a new subfolder in the cmsi387 directory (in this case, assignments).

USER

rm removes files or directories from the current directory.

Example: 

Edwards-MacBook-Air:cmsi387 Edward$ ls
README.md assignments homework  junk.txt
Edwards-MacBook-Air:cmsi387 Edward$ rm junk.txt
Edwards-MacBook-Air:cmsi387 Edward$ ls
README.md assignments homework

In this example, rm is called on junk.txt, which subsequently deletes the file without any output confirmation. ls shows that the file is indeed deleted when the command is invoked like so (NOTE: rm does not work when applied to folders unless invoked like so: rm -rf directory).

~

ssh utilizes the OpenSSH client to allow for the user to login to a remote machine and execute commands.

Example: 

Edwards-MacBook-Air:cmsi387 Edward$ ssh user@my.cs.lmu.edu
user@my.cs.lmu.edu's password:

ssh requires two entries: a valid username that has the correct permissions and the hostname of the remote machine (in this case, user and my.cs.lmu.edu). If entry is successful, the terminal will prompt for a password, and connect you to the remote host after entry.

||

su runs a Bash shell with "super user" permissions, allowing the user to change the current account associated with the running console.

Example: 

Edwards-MacBook-Air:cmsi387 Edward$ su
Password:
bash-3.2#

Invoking su prompts the user for a password (in order to prevent unauthorized access to superuser permissions). With correct entry, users have access to the virtual console with administrator privileges.

*

sudo allows a user with proper permissions to execute a command as the superuser (or, in some cases, another user).

Example: 

Edwards-MacBook-Air:cmsi387 Edward$ sudo bash
bash-3.2#

In this example, sudo bash is extremely similar to simply evoking su. With sudo, depending on the permission authority of your user account you may not even require a root password (as shown in the example above, account Edward is an administrator account).

!

uptime shows how long the system has been running.

Example: 

Edwards-MacBook-Air:tbramant-site Edward$ uptime
1:17  up 27 days,  9:29, 3 users, load averages: 2.57 2.19 2.05

When uptime is invoked, it returns:
  • The current time
  • How long the system has been running
  • How many users are currently logged on
  • the system load averages for the past 1/5/15 minutes.
